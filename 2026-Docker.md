## 1. 이미지(Image)를 파일로: Save & Load

주로 **이미지 전체(레이어 포함)**를 보존하고 싶을 때 사용. 이미지의 히스토리와 메타데이터가 그대로 유지.(EXPOSE, ENV 유지)

- **`docker save` (이미지 추출)**
  - 사용: 이미지 레포지토리를 `.tar` 파일로 저장
  - 명령어: `docker save -o [파일명].tar [이미지명]:[태그]`
  - *예시:* `docker save -o my_wordpress.tar wordpress:latest`
- **`docker load` (이미지 복원)**
  - 사용: `save`로 만든 `.tar` 파일을 다시 도커 이미지 목록으로 로드
  - 명령어: `docker load -i [파일명].tar`

------

## 2. 컨테이너(Container)를 파일로: Export & Import

실행 중이거나 멈춰있는 **특정 컨테이너의 현재 상태(파일 시스템)**를 스냅샷으로 찍을 때 사용합니다. 이미지의 레이어 이력은 사라지고 단일 파일 시스템으로 합쳐짐

- **`docker export` (컨테이너 추출)**
  - 사용: 컨테이너의 파일 시스템을 `.tar` 파일로 저장
  - 명령어: `docker export [컨테이너ID/이름] > [파일명].tar`
  - *예시:* `docker export my_db_container > db_backup.tar`
- **`docker import` (이미지로 변환)**
  - 사용: `export`된 파일을 가져와서 새로운 **이미지**를 생성
  - 명령어: `docker import [파일명].tar [새이미지명]:[태그]`

------

## 3. 주요 차이점 비교 테이블

| **구분**   | **Save / Load**                      | **Export / Import**                           |
| ---------- | ------------------------------------ | --------------------------------------------- |
| **대상**   | **이미지 (Image)**                   | **컨테이너 (Container)**                      |
| **내용물** | 이미지 레이어 + 메타데이터 (설정 등) | 컨테이너의 파일 시스템 스냅샷                 |
| **이력**   | 레이어 히스토리 보존 (용량 큼)       | 히스토리 삭제, 단일 레이어로 병합 (용량 작음) |
| **용도**   | 이미지 백업, 다른 서버로 이전        | 현재 실행 중인 상태를 이미지화                |



## 1. `docker cp` 기본 문법

기본적으로 `docker cp [원본] [대상]` 순서로 작성

### ① 호스트 → 컨테이너 (파일 넣기)

내 컴퓨터에 있는 설정 파일이나 소스를 컨테이너 내부로 보낼 때 사용

Bash

```
docker cp [호스트경로] [컨테이너이름]:[컨테이너내부경로]
```

- **예시:** 현재 폴더의 `index.html`을 `web_server` 컨테이너의 `/var/www/html`로 복사
  - `docker cp ./index.html web_server:/var/www/html/`

### ② 컨테이너 → 호스트 (파일 가져오기)

컨테이너 안에 생성된 로그 파일이나 설정 파일을 내 컴퓨터로 꺼낼 때 사용

Bash

```
docker cp [컨테이너이름]:[컨테이너내부경로] [호스트경로]
```

- **예시:** `db` 컨테이너의 로그 파일을 내 컴퓨터 현재 폴더로 가져오기
  - `docker cp db:/var/log/mysql/error.log ./`

------

## 2. 주요 특징(주의사항)

- **상태 무관:** 컨테이너가 **실행 중(Running)**일 때뿐만 아니라, **정지(Stopped)**된 상태에서도 복사가 가능
- **경로 자동 생성:** 복사하려는 대상 경로가 존재하지 않으면 자동으로 폴더를 생성하기도 하지만, 가급적 명확한 경로를 지정하는 것이 안전
- **권한 문제:** 복사된 파일의 소유권은 기본적으로 명령어를 실행한 사용자의 권한을 따르므로, 컨테이너 내부에서 권한 오류가 나면 `chown` 등으로 수정이 필요할 수 있음
- **와일드카드(\*) 미지원:** `*.txt` 같은 와일드카드는 직접적으로 지원 X. 폴더 전체를 복사하거나, 특정 파일을 하나씩 지정
